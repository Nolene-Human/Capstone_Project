Goals

In this assignment, my goal was to look at ways on how individuals can improve their SOHO network security.

I followed the Zero Trust model, deepened my understanding of the CyBOK Network Security principles through building a webserver and hosting my application on-prem. ü¶π

Step 1: Install Linux Ubuntu

  • Dual Boot Windows machine with Linux Ubuntu

  • Hardware setup and Troubleshooting
    üí¢ Windows initially did not allow dual-boot (or reboot Windows) until I entered the Bitlocker Key that I did not have.
    üí™ The solution was to enable SECURE BOOT again and run the following CLI command to find the key on the laptop. 

        manage-bde -protectors <the drive usually c>: -get
    Finding the solution took a few videos, but I finally got Bitlocker fully disabled. It is worth noting that Bitlocker is Windows additional data encryption security feature and disabling this opens up the machine to attacks. However, as demonstrated by Andy Malone MVP in How to HACK Windows Bitlocker - MUST SEE!, it is not hard to find this key and the convenience of having it off is currently exceeding the security benefit.

Step 2: Install Webserver

  • This projec is using NGINX over Apache because it is more resouce-effecient with higher performance and more secure. However Nginx cannot process dynamic content like Apache. Because the application hosted on this server is written using Flask Gunicorn had to be included into the stack.

    sudo apt install nginx

    sudo systemctl start nginx
Default FANG.html

  • Installing Gunicorn a Web Service Gateway Interface, this will act as my 'translater' between application and webserver.

    pip install gunicorn flask

  • Create simple 'Hello from Fang" flask application

  • Configure gunicorn config in nginx /etc/nginx/sites-available which NGINX will use to route Flask application using Gunicorn to webserver Nginx

sudo nano fang.conf


server {

    listen 80;

    location/{

            include pass http://unix:/home/servername/codefolder/******.sock; #folder where flask application is saved

            proxy pass http://127.0.0.1:8000; # this is where gunicorn is workers will launch
            proxy_set_header Host $host;

            }
}
üîî Reminder note : Remove the default config file created by NGINX to launch default html page üîî Reminder link config file to sites-enabled
